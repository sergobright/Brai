# Управляемая загрузка и установка APK

- Status: accepted
- Deciders: Владелец проекта
- Date: 2026-07-13
- Tags: архитектура, android, delivery, security

## Контекст

Передача APK в Android DownloadManager не даёт Brai надёжного пользовательского прогресса, повторного действия установки и контролируемой очистки. На реальном устройстве системная загрузка могла оставаться невидимой, хотя приложение видело запрос как активный. Одновременно `/dev-releases/` не был установлен в production Caddy, а Preview окружения сохраняли отличающийся release-пароль.

## Решение

- Brai самостоятельно скачивает APK своего установленного канала через HTTPS в закрытый `files/updates`, показывает фактические байты и проверяет `Content-Length` и SHA-256 из release index.
- После проверки Brai открывает штатный Android package installer через узкий FileProvider. Системное подтверждение остаётся обязательным; если разрешение на установку из этого источника отсутствует, открываются настройки приложения.
- Проверенный APK остаётся единственным готовым кандидатом, поэтому закрытый установщик можно повторно открыть кнопкой «Установить» без новой загрузки.
- После запуска версии с достигнутым target versionCode APK и частичные файлы удаляются. Прерванные и устаревшие кандидаты также очищаются.
- `/releases/` навсегда остаётся публичной страницей текущего Production APK. `/dev-releases/` маршрутизируется Caddy во всех окружениях, а Dev/Preview получают тот же `BRAI_RELEASE_PASSWORD`, что и Production.

## Рассмотренные альтернативы

- Оставить DownloadManager: меньше кода, но не решает наблюдаемый UX, повторную установку и управляемую очистку.
- Использовать PackageInstaller Session: даёт более низкоуровневый сеанс, но не отменяет системное подтверждение и добавляет состояние без пользы для текущего сценария.
- Хранить APK в публичных Downloads: файл виден пользователю, но Brai не может гарантировать очистку и накопление прекращается только вручную.

## Последствия

- Плюс: статус и процент соответствуют реально принятым приложением байтам.
- Плюс: скачанный APK можно установить повторно и он не накапливается после обновлений.
- Плюс: Android по-прежнему проверяет подпись пакета и показывает системное подтверждение.
- Минус: если процесс приложения будет принудительно завершён во время загрузки, частичный файл удалится, а загрузку потребуется повторить.
- Риск: Android может ограничить запуск установщика из фона; готовая кнопка «Установить» остаётся надёжным продолжением.

## Проверка

- Android unit tests проверяют адрес канала, дедупликацию, точные размер/SHA-256 и очистку после target versionCode.
- API tests проверяют checksum header, страницы, auth и rate limit.
- На Preview APK проверяются реальный процент, системный установщик, повторная установка после закрытия и очистка после обновления.
