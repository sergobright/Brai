# Публичная доставка APK и явное скачивание обновлений

- Status: accepted
- Deciders: Владелец проекта
- Date: 2026-07-13
- Tags: архитектура, android, delivery, security

## Контекст

Engine должен обнаруживать обновления в фоне, но скачивать их только после действия пользователя. Защищённая страница релизов не может быть единственной точкой скачивания APK: установленное приложение знает свой канал, но не имеет release-session. При этом полностью открытый каталог всех сборок и неограниченная раздача файлов создают лишнюю поверхность обнаружения и нагрузочный риск.

## Решение

- `/releases/` публично показывает только текущий Production APK; `/dev-releases/` под существующим паролем показывает все каналы.
- `/releases/download/:releaseKey` публично отдаёт актуальный APK известного канала, а legacy filename-маршрут публично сохраняется только для текущего Production файла.
- Один `RateLimiterMemory` ограничивает IP десятью начатыми APK-загрузками за 3600 секунд. `X-Forwarded-For` принимается только от loopback-соединения с локальным Caddy.
- Android `checkForUpdates()` только обнаруживает доступность. `downloadUpdate()` и `downloadApk()` являются отдельными явными командами.
- `downloadApk()` использует Android DownloadManager и установленный `nativeApkReleaseKey`; клиент без нового метода открывает прямой URL канала во внешнем браузере.

## Рассмотренные альтернативы

- Оставить все APK за release-паролем: установленное приложение не сможет обновляться без передачи технического пароля в клиент.
- Сделать публичным весь каталог: упрощает ручное скачивание, но раскрывает операционные Preview/Dev артефакты без необходимости.
- Добавить Redis-лимитер: даёт общий лимит нескольким процессам, но текущий API однопроцессный, поэтому инфраструктура преждевременна.
- Продолжать скачивание при проверке: расходует трафик без явного действия и смешивает независимые состояния UI.

## Последствия

- Плюс: приложение скачивает APK своего канала без секрета и без угадывания имени файла.
- Плюс: автоматические проверки не расходуют трафик архива, а Engine показывает однозначное действие.
- Минус: limiter сбрасывается при рестарте и не координируется между процессами.
- Риск: публичные прямые URL позволяют скачать Preview/Dev APK при знании ключа; каталог и имена при этом не публикуются.
- Риск: первый переходный APK не знает новый native bridge и один раз использует browser fallback.

## Проверка

- API-тесты проверяют видимость каталогов, auth cookie, канальную адресацию, заголовки, legacy 404 и одиннадцатый `429`.
- Android-тесты подтверждают отсутствие скачивания при discovery, дедупликацию DownloadManager и состояния результата.
- Client-тесты подтверждают тексты, действия, индикаторы, положение mobile overflow и отсутствие layout shift.
